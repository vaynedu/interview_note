---

## 为什么单独建这个目录

`14-projects/` 下有 `07-ecommerce-story.md` / `08-im-story.md` 等通用模板，但那是"虚构 + 行业最佳实践"。

**面试时讲虚构项目 → 被追问就崩**。

`personal/` 下放**你真实做过的项目**，脱敏后写：
- 公司名 → "X 公司" / 行业代号
- 业务名 → 抽象（"电商" / "IM" / "广告"）
- 数据 → 量级范围（"日均百万级"）
- 同事姓名 → "张三 / 李四" / 角色（"DBA / 架构师"）
- 技术栈细节 → 改用业界通用术语

## 写作建议

### 1. 选哪些项目写

参考 [../06-interview-star-template.md 第三章](../06-interview-star-template.md)（选项目 6 维度评分）。

**金牌项目特征**（任 4 项以上即写）：

- ✅ 有量化数据（QPS / 数据量 / GMV / 性能提升）
- ✅ 你是核心设计者 / 主 owner
- ✅ 有真实故障 + 复盘
- ✅ 有跨团队协作
- ✅ 有方案对比（不是被动跟着做）
- ✅ 完整闭环（设计 → 上线 → 运维 → 复盘）
- ✅ 近 1-2 年（5 年前的细节都模糊了）

### 2. 脱敏规则

| 内容 | 替换 |
| --- | --- |
| 公司名 | "X 公司" / "某电商公司" |
| 业务名 | 行业代号 ("电商" / "IM" / "广告") |
| 真实数字 | 量级范围 ("日均百万级 / 千万级") |
| 真实 GMV | "百万 / 千万 / 亿级" |
| 同事姓名 | 角色 ("DBA / 架构师 / TL") |
| 内部代号 | 业界通用术语 |
| 内部 RPC 框架名 | "类 gRPC / 类 Dubbo" |

### 3. 写作 5 步法

1. **先写时间线**（5 分钟）：列出项目从开始到结束的 5-10 个关键节点
2. **挑 3-5 个核心故事**（不是流水账）：每个故事独立完整
3. **每个故事按 STAR-L 写**（背景 / 任务 / 行动 / 结果 / 反思）
4. **量化 / 案例 / 取舍**：每个故事必含
5. **预演 6 个追问**（参考 [../07-ecommerce-story.md 追问应对预案](../07-ecommerce-story.md)）

### 4. 反模式

- ❌ 写成"我做了 A、B、C..."流水账（无重点）
- ❌ 全用通用模板的话术（缺乏真实感）
- ❌ 只讲成功不讲坑（不真实）
- ❌ "我们做了 X" 模糊"我们"（你的贡献呢？）
- ❌ 一篇写 3 个项目（每个都浅）

### 5. 维护节奏

- 每完成一个真实项目（或离职跳槽前），花 1-2 天整理一篇
- 面试前 1 周 review 一遍，针对性优化
- 每年大盘整理，删掉不再用的（5 年前的项目细节大多忘了）
