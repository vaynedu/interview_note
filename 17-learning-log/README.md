## 月度日志模板

参考 [2026-05.md](2026-05.md)。

每月 10 分钟回顾：
- 学了什么新东西
- 踩了什么坑
- 哪些知识点要补
- 哪些过时了要删
- 哪些读到的好文章 / 视频

## 面试反馈模板

`interviews/YYYY-MM-DD-<公司代号>.md`：

```markdown
# 面试反馈 - YYYY-MM-DD - <公司代号>

## 基本信息
- 公司: <代号，不写真名>
- 岗位: 资深 Go 后端 / Tech Lead
- 轮次: 一面 / 二面 / 主管面
- 时长: 60 min

## 题目清单（按问的顺序）
1. ...
2. ...

## 答得好的
- Q: ...
- 复盘: 为什么好（结构 / 案例 / 量化）

## 答得不好的
- Q: ...
- 当时答: ...
- 应该答: ...
- 进错题本: [wrong-answers.md](../../99-meta/wrong-answers.md)

## 面试官反馈（如有）
...

## 整体复盘
- 节奏 / 表达 / 心态
- 下次改进点
```

```markdown

（本季度面试 N 场，整体表现）

（新增 / 重构 / 删除的文件清单）

（具体到每月）
```

---

## 维护原则

- ❌ 不写宏大目标，写**具体记录**
- ❌ 不留空缺（缺点也要记）
- ✓ 每月初 / 月末 5-10 分钟即可
- ✓ 错题、面试反馈实时记，不要攒
- ✓ 季度复盘比月度日志更有价值，必做
