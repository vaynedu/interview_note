# AI 高频面试题

> 程序员面 AI，不一定要背模型训练细节，但要能讲清 LLM、RAG、Agent、工具调用、向量库、评估和工程落地。

## 一、LLM 基础题

### 1. LLM 为什么会幻觉？

答题要点：

- 模型生成的是概率上合理的 token。
- 它不天然验证事实。
- 上下文不足或问题歧义会放大幻觉。
- 训练数据可能过时。

解决：

- RAG。
- 工具调用。
- 引用来源。
- 测试验证。
- 人工 review。

### 2. 上下文窗口越大越好吗？

不是。

优点：

- 能放更多文档和代码。
- 更适合长任务。

代价：

- 成本更高。
- 延迟更大。
- 噪声更多。
- 模型可能忽略中间信息。

关键是上下文质量，而不是越长越好。

## 二、RAG 题

### RAG 流程是什么？

```text
文档切块 -> embedding -> 向量库 -> 检索 -> rerank -> 放入 prompt -> LLM 回答
```

### RAG 有什么坑？

- 文档切块太粗或太细。
- 检索召回差。
- 文档过时。
- 权限过滤缺失。
- 模型没有基于来源回答。
- 只做向量检索，不做关键词和重排。

## 三、Agent 题

### Agent 和 ChatBot 区别？

ChatBot：

- 一问一答。
- 主要生成文本。

Agent：

- 有目标。
- 能拆任务。
- 能调用工具。
- 能观察结果。
- 能迭代修正。

### Agent 为什么需要工具？

因为模型本身不能：

- 读取你的实时数据库。
- 知道最新网页。
- 修改文件。
- 执行测试。
- 调内部 API。

工具让 Agent 能观察和行动。

## 四、Coding Agent 题

### 如何让 AI 更好地写代码？

要给：

- 相关文件。
- 项目规范。
- 明确范围。
- 测试命令。
- 完成标准。

不要：

- 给模糊大任务。
- 不让它读代码。
- 不让它跑测试。
- 不 review 输出。

### AI 写代码最大的风险是什么？

- 幻觉 API。
- 误解业务。
- 改动范围过大。
- 没处理失败路径。
- 没有测试。
- 安全和权限问题。

## 五、向量库题

### 向量库解决什么？

向量库用于相似度检索：

- 文档问答。
- 语义搜索。
- 推荐召回。
- 代码检索。

但它不负责：

- 事务。
- 强一致。
- 权限语义。
- 复杂推理。

## 六、评估题

### 如何评估一个 AI 应用？

维度：

- 准确率。
- 召回率。
- 幻觉率。
- 工具调用成功率。
- 端到端任务完成率。
- 延迟和成本。
- 安全违规率。
- 用户满意度。

Coding Agent 还要看：

- 测试通过率。
- 代码 review 问题数。
- 回归 bug。
- 人工修改比例。

## 七、安全题

常见风险：

- Prompt injection。
- 数据泄漏。
- 工具误操作。
- 过度授权。
- 生成不安全代码。
- 依赖投毒。

治理：

- 权限最小化。
- 工具审批。
- 敏感信息脱敏。
- 日志审计。
- 输出验证。
- 人工 review。

## 八、面试表达模板

```text
我会把 AI 应用看成模型、上下文、工具和评估四部分。
模型负责生成和推理，上下文决定它能看到什么，工具让它连接真实系统，评估负责判断结果是否可靠。
RAG 适合知识问答，但依赖文档质量和检索质量；
Agent 适合多步任务，但必须控制权限、记录执行过程，并用测试或指标验证。
```
